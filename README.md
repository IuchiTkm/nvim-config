## キーマップ一覧

### ファイルツリー
| キー | 動作 |
|------|------|
| `Ctrl+b` | ファイルツリーの開閉 |

### タブ（バッファ）
| キー | 動作 |
|------|------|
| `Shift+l` | 右のタブへ |
| `Shift+h` | 左のタブへ |
| `\w` | 現在のタブを閉じる |

### ウィンドウ分割
| キー | 動作 |
|------|------|
| `\d` | 上下分割 |
| `\r` | 左右分割 |
| `\x` | ウィンドウを閉じる |
| `\h` / `\j` / `\k` / `\l` | ウィンドウ間の移動 |
| `\z` | ズーム切り替え |

### Copilot
| キー | 動作 |
|------|------|
| `\ccq` | バッファ内容を使ってCopilotにチャット |

### Markdown
| キー | 動作 |
|------|------|
| `\mp` | Markdownプレビューのオン/オフ |

### C#（.cs ファイルのみ）
| キー | 動作 |
|------|------|
| `gd` | 定義へジャンプ |
| `gr` | 参照一覧を表示（`q` で閉じる） |
| `\rn` | シンボルのリネーム（プロジェクト全体で一括置換） |
| `K` | ホバードキュメント表示 |
| `Tab` / `Shift+Tab` | 補完候補を選択 |
| `Enter` | 補完候補を確定 |
| `Ctrl+o` | ジャンプ前の場所に戻る |
| `\f` | LSPフォーマット（手動） |
| `\nd` | XMLサマリー自動生成 |

### lazygit
| キー | 動作 |
|------|------|
| `\gg` | lazygit を開く |

### ターミナル
| キー | 動作 |
|------|------|
| `\t` | ターミナルを開く/閉じる |
| `Ctrl+k` | ターミナルからコードペインへ移動 |
| `\j` | コードからターミナルペインへ移動 |

## プラグイン詳細

### nvim-tree

ファイルツリーを右側に表示するプラグイン。

**ツリー内の操作**

| キー | 動作 |
|------|------|
| `Enter` / `o` | ファイルを現在のウィンドウで開く |
| `t` | 新しいタブで開く |
| `T` | 新しいタブで開く（フォーカスはツリーのまま） |
| `a` | 新規ファイル/ディレクトリ作成（末尾に `/` でディレクトリ） |
| `r` | リネーム / 別フォルダへ移動（パスごと書き換え） |
| `d` | 削除 |
| `x` | カット |
| `c` | コピー |
| `p` | ペースト |
| `R` | ツリーを更新 |
| `H` | 隠しファイルの表示切り替え |
| `q` | ツリーを閉じる |

---

### bufferline

開いているバッファをVSCodeのようにタブ表示するプラグイン。

| キー | 動作 |
|------|------|
| `Shift+l` | 右のタブへ |
| `Shift+h` | 左のタブへ |
| `\w` | 現在のタブを閉じる |

---

### lualine

ステータスラインを見やすく表示するプラグイン。設定不要で自動的に有効になる。

---

### coc.nvim

LSP（Language Server Protocol）による補完・定義ジャンプなどを提供するプラグイン。

**主なコマンド**

| コマンド | 動作 |
|----------|------|
| `:CocInstall coc-<name>` | 言語サーバーのインストール |
| `:CocList extensions` | インストール済み拡張の一覧 |
| `:CocUpdate` | 拡張の更新 |

**主なキー（デフォルト）**

| キー | 動作 |
|------|------|
| `gd` | 定義へジャンプ |
| `gr` | 参照一覧 |
| `K` | ホバードキュメント表示 |
| `Tab` | 補完候補を選択 |

---

### CopilotChat

| キー | 動作 |

| `\ccq` | バッファ内容を添付してチャット入力 |

**主なコマンド**

| コマンド | 動作 |
|----------|------|
| `:CopilotChat` | チャットウィンドウを開く |
| `:CopilotChatClose` | チャットウィンドウを閉じる |
| `:CopilotChatReset` | チャット履歴をリセット |

### render-markdown

| キー | 動作 |
|------|------|
| `\mp` | プレビューのオン/オフ切り替え |

### winresizer

`Ctrl+e` でウィンドウリサイズモードに入るプラグイン。

| キー | 動作 |
|------|------|
| `Ctrl+e` | リサイズモード開始 |
| `h` / `j` / `k` / `l` | ウィンドウ幅・高さを変更 |

### lazygit.nvim

Git の操作を TUI で行えるプラグイン。別途 lazygit 本体のインストールが必要。

**インストール（初回のみ）**

```powershell
winget install JesseDuffield.lazygit
```

| キー | 動作 |
|------|------|
| `\gg` | lazygit を開く |

**lazygit 内の主な操作**

| キー | 動作 |
|------|------|
| `?` | ヘルプ表示 |
| `q` | lazygit を閉じる |
| `Space` | ファイルのステージ/アンステージ |
| `c` | コミット |
| `p` | プッシュ |
| `P` | プル |

### toggleterm.nvim

画面下部にターミナルを表示するプラグイン。ターミナルを開いたままコードを編集できる。

| キー | 動作 |
|------|------|

### C# サポート

`.cs` ファイルを開いたときのみ有効になる C# 専用の LSP・補完設定。`lua/plugins/csharp.lua` で管理。

**構成**

| コンポーネント | 役割 |
|---|---|
| `csharp-ls` | C# 言語サーバー（dotnet グローバルツール） |
| `nvim-cmp` | 補完エンジン |
| `mason.nvim` | 言語サーバー管理 |

**初回セットアップ（インストール済みの場合は不要）**

```powershell
dotnet tool install --global csharp-ls --version 0.20.0
```

> csharp-ls 0.21.0 以降は .NET 10 が必要なため、.NET 9 環境では 0.20.0 を使用する。

**注意事項**

- プロジェクトルート（`.sln` または `.csproj` があるディレクトリ）から Neovim を起動すること
- coc.nvim は `.cs` ファイルでは自動的に無効化される

### vscode.nvim

VSCode Dark+テーマをベースにしたカラースキーム。VS2022のダークテーマに近い配色を提供する。

- キーワード：青
- 型・クラス名：水色
- 文字列：オレンジ
- コメント：緑（イタリック）
- メソッド名：黄色

### nvim-treesitter

構文解析によって正確なシンタックスハイライトを提供するプラグイン。C#パーサーを含む。

初回起動時に自動でC#パーサーがインストールされる。

---

### lsp_signature.nvim

関数を呼び出すときに、引数のシグネチャをリアルタイムでポップアップ表示するプラグイン。LSP（csharp-ls）と連携して動作する。

```
void Console.WriteLine(string value)
                       ^^^^^^^^^^^^  ← 現在入力中の引数をハイライト
```

---

### neogen

メソッド・クラス・プロパティの上で `\nd` を押すと、C# のXMLドキュメントコメントを自動生成するプラグイン。

| キー | 動作 |
|---|---|
| `\nd` | カーソル位置のXMLサマリーを生成 |

生成例（メソッド上で実行）：

```csharp
/// <summary>
/// 
/// </summary>
/// <param name="id"></param>
/// <returns></returns>
public string GetUser(int id) { ... }
```

---

### tobira.nvim

実際の編集習慣を観察して、より効率的なVimコマンドを提案してくれる学習支援プラグイン。148コマンド・34の検出パターンに対応し、約100回の使用で提案が止まる。

**コマンド**

| コマンド | 動作 |
|---|---|
| `:Tobira` | 今すぐ提案を表示 |
| `:TobiraGuide` | チートシート表示 |
| `:TobiraProgress` | 習得度確認 |
| `:TobiraStats` | 使用統計確認 |
| `:TobiraReset` | 使用データをリセット |

---

### hlchunk.nvim

インデントのガイドラインと現在のスコープ（ブロック）をハイライト表示するプラグイン。

- **chunk**: 現在カーソルがあるブロックの範囲をハイライト
- **indent**: インデントガイドラインを表示

---

### nvim-treesitter-context

画面上部に現在いるクラス・メソッドのシグネチャを常時表示するプラグイン。長いファイルでスクロールしても自分がどのメソッド内にいるか確認できる。

- 最大3行まで表示
- `-` で本文と区切り

**コマンド**

| コマンド | 動作 |
|---|---|
| `:TSContext enable` | 有効化 |
| `:TSContext disable` | 無効化 |
| `:TSContext toggle` | 切り替え |

## プラグインの追加手順

1. `lua/plugins/` に新しい `.lua` ファイルを作成する（例: `lua/plugins/foo.lua`）
2. 以下の形式で設定を書く

```lua
return {
    "author/plugin-name",
    dependencies = { "依存プラグイン（不要なら省略）" },
    keys = {
        { "<leader>xx", "<cmd>コマンド<CR>", desc = "説明" },
    },
    config = function()
        -- 追加設定が必要な場合はここに書く
    end,
}
```

3. Neovim を再起動するか `:Lazy sync` を実行してインストール
4. キーマップを追加した場合は `lua/core/keymaps.lua` ではなく、上記のように `keys` にまとめて書くのが推奨

> `keys` に書いたプラグインは、そのキーを押すまで読み込まれない（遅延ロード）のでパフォーマンスに優れる。
